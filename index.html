Introduction

I've just finished a project using Backbone and I would like to share a few tips, tricks, and libraries that helped me create a non-trivial web-client for an existing API.

I think of Backbone as a Minimal Viable Framework. Backbone has support for a lot of functionality but as an application developer, you will often have to code the last mile or pick and choose from what is available in backbone to create your ideal framework. If you're starting a completely new application, you'll be able to tailor your back-end to suite a backbone front-end. That's awesome and you'll likely find yourself building a more RESTful api than you might have otherwise because backbone benefits from that style. However, building a new API is often a luxury because you'll probably be building a front-end for an existing API.
